How to use MIDI→GRAB
STEP 1 — BUILD

Drop your .mid file, set a title in the Level Metadata panel, adjust settings, then press ⚡ Build Level. Press ⬇ .level to download directly as a .level file.

STEP 2 — ADD TO GRAB (PC / STEAM)

Copy the .level file into:
Documents/GRAB/levels/user/
The level appears in your level list next time you open the in-game editor.

STEP 2 — ADD TO GRAB (QUEST)

Use CX File Explorer, SideQuest, or ADB. Copy to:
Android/data/com.slindev.grab
/files/levels/user/

OPTIONAL — EDIT IN GRAB TOOLS

For deeper edits, go to grab-tools.live/editor, import your .json. Add platforms, checkpoints, signs, and more — then export a final .level.

TRACK VISIBILITY
Creates a separate instrument layer for each MIDI track — each track gets its own set of pitch-reuse sound blocks stacked vertically in the level. Sound parameters are matched from a curated preset library by instrument name (piano, strings, brass, drums, etc.), or generated via the GM map if no match is found. Frequency is always set to the correct note pitch. Wave type selector is disabled.
When the same pitch plays twice simultaneously, creates a second sound + trigger block for it so both fire independently. May not work perfectly on all MIDIs.
Animated blocks stay inside triggers for the exact note duration. The sound repeats while inside and stops when the block exits. Attack and release are disabled for seamless looping. Default sound length is set to ~1 second per repeat cycle.
Reuses one animated trigger per unique pitch with a single static sensor block. Repeated notes in the same pitch lane are merged into the same trigger timeline, so the saver actually cuts node churn. With Exact Length, triggers stay inside the sensor block for the note's full duration and only move in Z (never left/right).
Recommended: 15–20 blocks for best performance. Higher counts add more simultaneous note support but increase level complexity and load time.
